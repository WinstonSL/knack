# Knack 项目铁律（给 AI 看）

本目录下任何文件改动都必须遵守以下规则。**违反即返工**。

## 0. 起手动作

动手之前，先读：
1. 本文件（AGENTS.md）—— 铁律
2. `README.md` —— 当前最新版快照
3. 必要时翻 `CHANGELOG.md` —— 历史版本变更

## 1. 命名与版本（2026-06-11 更新，三位数制）

从 v2.3 之后，**所有新版本一律采用三位数 `vX.Y.Z`**：

- **每次改动都必须新建文件，命名 `Knack-vX.Y.Z.html`**。绝对不允许覆盖旧版。
- **递进规则**：每次普通迭代仅 `Z` +1。
  - v2.3 之后下一版 = `Knack-v2.3.1.html`
  - 再下一版 = `Knack-v2.3.2.html`
  - …… 一直到 `Knack-v2.3.9.html`
- **进位规则**：`Z` 在 9 之后**不进 10**，而是 `Y` +1，`Z` 归 0：
  - v2.3.9 → 下一版 = `Knack-v2.4.0.html`
  - v2.4.9 → 下一版 = `Knack-v2.5.0.html`
- **大版本（X +1）**仅用于破坏性改动 / 重大重构 / 全面新功能，且必须由用户明确说"升大版本"才能用。
  - 例：用户说 "升 v3"，才出 `Knack-v3.0.0.html`。
- **唯一例外（覆盖旧版）**：用户**明确授权**说"覆盖 vX.Y.Z 不升版"才能原地改文件。这种情况要在 CHANGELOG 当前版段落 Notes 里注明「同日补丁，未升版（用户明确授权）」。**不写 Notes 等于违规。**
- 旧版本搬到 `archive/` 子目录，根目录只保留**当前最新**那一个 `Knack-vX.Y.Z.html` 作为入口。

### 何时 +Z、何时 +Y、何时 +X（速判表）

| 改动性质 | 操作 | 例子 |
|---|---|---|
| 修 bug / 调样式 / 加小功能 | `Z+1` | v2.3 → v2.3.1 |
| `Z` 已经到 9 还要继续迭代 | `Y+1, Z=0` | v2.3.9 → v2.4.0 |
| 重大重构 / 破坏性改动（用户授权） | `X+1, Y=0, Z=0` | v2.x → v3.0.0 |
| 用户授权"覆盖不升版" | 不动文件名 | 在 CHANGELOG Notes 里写明 |

## 2. 单文件 + 无外部依赖

- 必须单文件 HTML，浏览器双击即用。
- 不引入任何 CDN、不引入任何 npm 依赖。
- Logo / 图标 / 样式 / 脚本全部内联在 HTML 内。
- `assets/` 目录只用来放给宣传用的素材副本（如 SVG），**不被运行时引用**。

## 3. 文档同步（每次升版必做）

发布新版本时，三件事一起做：

1. 新建 `Knack-vX.Y.Z.html`（不覆盖）
2. 更新 `README.md` 的「当前最新版」「v?.?.? 能力快照」「关键代码位置」段
3. 在 `CHANGELOG.md` **顶部**追加一段（不要改老版本段落）
4. 把上一版搬进 `archive/`（如果根目录已经有 ≥2 个版本文件）

## 4. 用户偏好（写代码 / 写文档时遵守）

- 中文交流，活泼可爱、好读好记。
- 不要花式辞藻 / 不要打鸡血 / 不要无意义客套。
- 文件命名优先用品牌名（`Knack-vX.Y.Z.html`）或中文名。
- HTML 风格：scroll-triggered 淡入、左右对齐、单行展示、不靠扩字号填空白。

## 5. 关键路径速查

- 入口（最新）：`Knack-v2.3.html`（下一次迭代起命名变为 `Knack-v2.3.1.html`）
- 历史版本：`archive/`
- 项目说明：`README.md`
- 变更日志：`CHANGELOG.md`
- Logo 素材：`assets/knack-logo.svg`

## 6. 不要做的事

- ❌ 不要把品牌名改回"HTML 修改器"
- ❌ 不要把单文件拆成多文件 / 引外部依赖
- ❌ 不要在不升版本号的情况下原地改老文件
- ❌ 不要把 `Z=9` 直接进位成 `Z=10`，必须 `Y+1, Z=0`
- ❌ 不要擅自跳大版本（X+1）—— 必须用户明确授权
- ❌ 不要在 README 里堆历史版本（那是 CHANGELOG 的事）
- ❌ 不要悄悄删旧版本，必须搬到 archive/
